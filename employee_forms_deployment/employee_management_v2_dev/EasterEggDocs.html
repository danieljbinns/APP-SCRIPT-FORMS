01 Overview

The Easter Egg system is a hidden, client-side overlay injected into every page of the DEV project. It provides 10 fully-themed UI skins — each replacing colors, fonts, borders, animations, audio, and UI text with universe-specific styles — activated by secret keyboard sequences, without any server-side changes.

The system is entirely self-contained in a single file (EasterEgg.html) and included at the bottom of every dev page via:

<?!= include('EasterEgg') ?>
⚠ Scope Warning
This file must never be included in staging or prod folders. It is DEV-only by convention and the DEV ONLY comment at line 1.

What it does

  • Listens for typed keyboard sequences and the Konami code — no UI affordance is visible until triggered
  • Injects a DOM subtree (#eg-root) and adds CSS classes to <body> to restyle the entire page
  • Loads live stats from the spreadsheet via google.script.run
  • Plays ambient audio using the Web Audio API and speech synthesis greetings
  • Persists active state across page loads via localStorage
  • Restores all changes cleanly on deactivation — no page reload needed
02 Architecture

Inclusion Pattern

The file is a GAS template fragment (no <!DOCTYPE> — just a <style> and <script> block). It lands at the end of the page DOM via the GAS include() helper.

/* In any dev .html file — bottom of <body> */
<?!= include('EasterEgg') ?>

IIFE Guard

The entire JS block is wrapped in an IIFE that bails if #eg-root already exists, preventing double-injection if any page includes the fragment more than once.

(function(){
  'use strict';
  if(document.getElementById('eg-root'))return;
  /* ... all egg code ... */
})();

ES5 Constraint

All JavaScript is written in strict ES5 — no const, let, arrow functions, or template literals. This is required because GAS template HTML is served without a transpiler and must run in older embedded WebViews.

Click the site logo (any .logo, #logo, header img, .brand-logo, or #header-logo element) 5 times within 2.5 seconds. Identical result to Konami — opens the admin panel.

Theme Switching While Active

Typing a new code while another theme is already active triggers a smart switch:

  • Same family (geo→geo or sci-fi→sci-fi): calls stSwitchTheme() — fast CSS-only swap, no DOM rebuild
  • Cross-family (geo→sci-fi or vice versa): calls deactivate(true) then activate(newTheme) — full DOM rebuild because the two families use completely different HTML structures

05 CSS Architecture

Body Class Strategy

The easter egg uses a cascade of body classes. The key selector trick is body.eg:not(.t-geo) — this single rule applies sci-fi component overrides to all 9 non-geo themes without listing each one. Individual themes then override specifics using body.eg.t-{name}.

ClassPurposeApplied by
body.egMaster switch — gates all egg CSS. Font scale variables.activate()
body.eg.t-{name}Per-theme CSS token values (--st-bg, --st-mid, etc.) and visual overridesstSwitchTheme() / activate()
body.eg:not(.t-geo)Sci-fi component overrides for all non-geo themes — headers, cards, inputs, tables, buttonsCSS rule (passive)
body.eg.is-lcarsLCARS-style modifier: left-bar cards, rounded controls, gold ticker — used by TNG, DS9, VOYstSwitchTheme() when d.lcars===true
body.eg.sc-{sm/md/lg/xl}Font scale — remaps all --fs-* variablesstSetScale()

CSS Token System

Each sci-fi theme defines 14 CSS custom properties on its body.eg.t-{name} selector. The :not(.t-geo) component rules consume these tokens exclusively — so adding a new sci-fi theme only requires one new token block.

/* Example — TOS token block */
body.eg.t-tos {
  --st-bg: #040200;   /* page background */
  --st-panel: #090500; --st-panel2: #0E0700;  /* card / header */
  --st-hi: #FFCC55;   /* primary text */
  --st-mid: #FFAA00;  /* accent / borders */
  --st-lo: #CC8800;   /* secondary accent */
  --st-dim: #886600;  /* muted text */
  --st-faint: #3A2200; /* subtle borders */
  --st-red: #FF3300; --st-green: #00DD55;
  --st-blue: #2266DD; --st-cyan: #00BBDD;
  --st-fd: 'VT323','Courier New',monospace;  /* display font */
  --st-fb: 'Share Tech Mono','Courier New',monospace; /* body font */
  --st-btn-r: 0px;    /* button border-radius */
}

Geo Theme Exception

The geo theme is a full override — it sets app-level CSS variables (--bg-color, --card-bg, etc.) rather than the --st-* tokens, and uses body.eg.t-geo selectors with higher specificity than the :not(.t-geo) rules. This is why switching between geo and sci-fi requires a full DOM rebuild.

Font Scale System

All font sizes reference --fs-* variables. Four scale classes remap these on body.eg. The control bar has S/M/L/XL buttons calling stSetScale().

08 Audio System

Audio is optional and off by default. The 🔇 OFF / 🔊 ON button in the control bar toggles it. On enable, the AudioContext is created (satisfying browser autoplay policy — requires a user gesture), ambient sounds start, and a speech synthesis greeting plays.

ⓘ Browser Note
Geo's chiptune music has its own separate toggle (#geo-music-btn). The #st-ab ambient toggle exists only on sci-fi/galpha themes.

Ambient Sound Functions

playTOS()
OscillatorNode — sine
Random freq from [440, 660, 880, 1100] Hz — short 180ms ping. Interval 2–5s.
playLCARS()
OscillatorNode — sine, ramp
500→1400 Hz ramp in 150ms. LCARS "chirp." Used by TNG, DS9, VOY. Interval 4–7s.
playBSG()
OscillatorNode — square ×2
Two square wave pulses (330 Hz, 440 Hz) 220ms apart — DRADIS ping. Interval 2.5–4.5s.
playMatrix()
BufferSource — noise
White noise burst through 900 Hz bandpass filter — "digital glitch." Interval 0.8–2s.
playWho()
OscillatorNode — sawtooth
Sawtooth wave descending from ~700 Hz with alternating ±90 Hz wobble over 2.2s — TARDIS dematerialisation. Interval 5–9s.
playOffice()
OscillatorNode — sine ×5
G4-A4-G4-F4-G4 acoustic guitar feel — 5 notes at 180ms intervals. Interval 6–10s.
playGalpha()
OscillatorNode — sine ×8
Sparkly ascending arpeggio A5→C6→E6→G6→E6→C6→G6→C7 at 100ms per note. Interval 1.5–3.5s.
Geo Chiptune
OscillatorNode — square, looping
24-note GEO_NOTES array — C5 pentatonic melody played on square wave oscillators, looping via setTimeout scheduling.

09 Stats System

Stats are fetched once per activation via google.script.run.getStarTrekStats() and cached in statsData. Re-fetching only happens if the stats panel is opened and statsLoaded === false.

Data Flow

Server Function — StarTrekTOSStats.js

getStarTrekStats() reads the DASHBOARD_VIEW sheet and ACTION_ITEMS sheet, returning a JSON object with totals, top requesters, top sites, top managers, best month, and last updated timestamp.

Stats Render — Sci-Fi (renderTrekStats)

Uses the current theme's T[theme] data object for field labels (d.s1–d.s8, d.vlabel, d.vsub, d.l1–d.l3). The same data renders differently in every theme with themed vocabulary.

Stats Render — Geo (renderGeoStats)

Populates the inline #geo-stats-panel-wrap: animated visitor counter, 12-stat grid, 3-column leaderboard (requesters/sites/managers), best month banner, and dynamically awarded achievement badges.

Offline Fallback

If typeof google === 'undefined' (dev preview, no GAS context), the render shows OFFLINE — NO SERVER CONNECTION (sci-fi) or renders the geo panel with an error state.

10 State & Persistence

localStorage Key: tg_eg_v1

When the egg is active, the current state is written to localStorage on every theme/scale/audio change. On page load, the IIFE reads this and re-activates automatically if saved.on === true.

{
  "on": true,
  "theme": "tng",         // active theme id
  "scale": "sc-md",        // active scale class
  "audio": false           // audio state — NOT auto-restored (requires gesture)
}
♦ Audio on Restore
Audio state is stored but not automatically re-enabled on page reload — Web Audio requires a user gesture to resume. The stored value is passed to activate() as a hint for future use.

  • Theme button click — activates that theme (smart cross-family switching)

12 Function Reference
Function
Description
activate(theme, scale, audio)
Main entry point. Adds body.eg, creates #eg-root, builds geo or trek UI, wires all event listeners, calls swapStatus/swapSubmit, saves pref.
deactivate(silent)
Tears down everything: stops audio, removes #eg-root, strips all body classes, restores text swaps, clears localStorage. If silent=true, skips the exit toast.
stSwitchTheme(name)
Fast CSS-only theme switch within the sci-fi family. Updates body classes, all injected UI text strings, re-renders stats if visible, handles audio.
stSetScale(sc)
Applies a font scale class to body. Updates control bar button states.
openAdmin() / closeAdmin()
Creates/removes the #eg-admin overlay. openAdmin reads THEME_META and current active state to build the panel.
fetchStats()
Guards against double-fetch with statsLoaded. Calls getStarTrekStats() via google.script.run. Routes result to renderTrekStats or renderGeoStats.
renderTrekStats(data, d)
Builds the stats panel HTML string for sci-fi themes. Uses d (theme config) for all labels. Returns HTML, does not mutate DOM.
renderGeoStats(data)
Populates the inline geo stats panel: visitor counter, 12-box stat grid, 3 leaderboards, best month, achievement badges.
swapStatus(theme) / swapSubmit(theme)
DOM walks to replace status badge text and submit button text using STATUS_MAPS / SUBMIT_MAPS. Saves originals in data-eg-orig.
restoreSwaps()
Reverses all data-eg-orig substitutions.
stToggleAudio()
Initialises AudioContext on first call (satisfies browser gesture requirement), toggles ambient loop and speech greeting. Updates #st-ab button state.
startAmbient() / stopAmbient()
Starts/stops the ambient sound tick loop using SND[curTheme] and INT[curTheme] for the sound function and interval range.
speakGreeting(theme)
Fires a SpeechSynthesisUtterance with theme-specific rate, pitch, and greeting text from T[theme].greeting.
startRain() / stopRain()
Matrix canvas rain animation using setInterval. startRain re-sizes canvas to viewport. Window resize listener re-fires startRain when Matrix is active.
toggleGeoMusic() / stopGeoMusic()
Toggles the GEO_NOTES chiptune loop. Uses AudioContext scheduling via playGeoNotes() — all 24 notes pre-scheduled per loop iteration.
attachLogoHooks()
Queries 6 logo selectors and adds click listeners for the 5×logo admin shortcut. Called once at IIFE init.
savePref() / loadPref() / clearPref()
localStorage read/write/delete for key tg_eg_v1. Wrapped in try/catch for incognito mode safety.
toast(msg)
Shows a fixed-position notification for 4.5 seconds. Replaces any existing toast. Styled differently per theme via the active body class.
startGeoTitleBlink() / stopGeoTitleBlink()
Cycles document.title through GEO_TITLES array every 1.8 seconds while geo theme is active.
