Employee Setup Form (Form 2)

Form Purpose:

This is the first form after InitialRequest. It handles core employee identity and access setup.

Fields Include:

  • Internal Employee ID - Auto-generated 6-digit number, editable
  • SiteDocs Setup (All Employees):
    • Worker ID
    • Job Code
  • SiteDocs Admin/Login Access (Conditional - if checkbox selected):
    • Training Username
    • Training Password
  • DSS Training Access (All Employees):
    • Username (auto-generated: FirstInitialLastName-SiteCode-YYYYMMDD)
    • Password
  • BOSS System Access:
    • WIS Module Assignment
  • Setup Notes

Data Passed from InitialRequest (Form 1):

  • Employee Name, Hire Date, Position
  • Site Name, Site Code, Manager Info
  • Employment Type
  • SiteDocs Access flag (determines if admin fields show)
  • Email Requested

Auto-Generated/Pre-filled Fields:

  • Internal Employee ID (6-digit number)
  • DSS Username (FirstInitialLastName-SiteCode-YYYYMMDD)

Workflow Position:

Sequential: Always triggers immediately after InitialRequest (Form 1)

Next: After completion, triggers parallel forms (IT, Jonas, DSS Learning, etc.)
